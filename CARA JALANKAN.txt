# PANDUAN MENJALANKAN PROJECT KOPMASSHOP (Versi Update)

Project ini telah dioptimasi untuk berjalan menggunakan **Python 3.11** dan **Django 4.2 (LTS)**.

## 1. Persiapan Virtual Environment
Jika Anda memulai dari awal, buat dan aktifkan virtual environment:
```powershell
# Jika belum ada venv:
python -m venv .venv

# Aktifkan venv (Windows):
.\.venv\Scripts\activate
```

## 2. Instalasi Dependencies
Pastikan library yang terinstal adalah versi terbaru yang kompatibel:
```powershell
pip install -r requirements.txt
```

## 3. Konfigurasi Environment Variable (.env)
1. Buka file `.env`.
2. Pastikan konfigurasi database sudah benar (port default PostgreSQL adalah 5432, namun project ini dikonfigurasi menggunakan port 5555 sesuai environment Anda).
3. Update `ADMIN_EMAIL` dan `ADMIN_PASSWORD` sesuai keinginan untuk akun admin utama.

## 4. Migrasi Database
Jalankan migrasi untuk membuat tabel baru dengan prefix `kopmas_`:
```powershell
python manage.py makemigrations
python manage.py migrate
```

## 5. Seed Data (Roles & Akun Admin)
Jalankan perintah seeder untuk mengisi role (Admin, Seller, Buyer) dan membuat akun administrator utama yang diambil dari file .env:
```powershell
python manage.py seed_roles --with-admin
```
*Catatan: Akun admin yang dibuat otomatis memiliki status Superuser.*

## 6. Menjalankan Server
Jalankan server pengembangan Django:
```powershell
python manage.py runserver
```
Akses melalui browser di: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 7. Troubleshooting
- **IntegrityError / Table Missing**: Pastikan Anda sudah menjalankan `migrate` dan `seed_roles` setelah melakukan perubahan pada `models.py`.
- **Google Login Error**: Jika terjadi error saat login Google, coba restart server (`Ctrl+C` lalu jalankan `runserver` lagi) agar metadata skema tabel yang baru termuat dengan benar.

---
**Teknologi Utama:** Django 4.2.x, Python 3.11, PostgreSQL.
